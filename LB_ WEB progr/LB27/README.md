# Лабораторна робота №27: Throttle, Debounce та Lazyload

## Виконані завдання:
1. **Throttle:** Реалізовано функцію обмеження частоти викликів для події прокрутки (`scroll`). Поточне значення прокрутки оновлюється в інтерфейсі не частіше ніж раз на 300 мс, що суттєво зменшує навантаження на процесор.
2. **Debounce:** Налаштовано затримку виконання для поля пошуку. Обробка введеного тексту відбувається лише через 500 мс після повної зупинки введення, що дозволяє уникнути зайвих запитів при кожному натисканні клавіші.
3. **Lazy Load:** Впроваджено технологію відкладеного завантаження зображень через `Intersection Observer API`. Зображення завантажуються лише в момент потрапляння у видиму частину екрана, що пришвидшує початкове завантаження сторінки.

## Як перевірити:
1. Відкрийте файл **index.html** у браузері.
2. Відкрийте консоль розробника (Права кнопка миші -> **Переглянути код елемента** -> вкладка **Console**).
3. **Throttle:** Почніть прокручувати сторінку — в консолі з'являтимуться логи з координатами, а в шапці сторінки оновлюватиметься статус.
4. **Debounce:** Введіть текст у поле пошуку — результат з'явиться в консолі лише після паузи.
5. **Lazyload:** Прокрутіть до нижньої частини сторінки — ви побачите, як зображення з'являються та завантажуються динамічно.
